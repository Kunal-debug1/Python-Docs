Introduction to Python

Beginner ⏱ ~5 min read

Python is a high-level, interpreted programming language created by Guido van Rossum and released in 1991. It emphasizes code readability and simplicity, which makes it one of the most beginner-friendly languages available today — while also being powerful enough to drive cutting-edge applications.

What is Python?

At its core, Python is a tool for telling computers what to do. Unlike lower-level languages, Python code looks almost like plain English, which means you can focus on solving problems rather than fighting with syntax.

ℹ️
Note

Python is an interpreted language — code runs line by line, making it easy to test and debug interactively.

Why choose Python?

Python's philosophy can be summed up in a few ideas: code should be readable, simple should beat complex, and practical problems deserve practical solutions. These ideas made Python the #1 language in fields like data science, AI, web development, and automation.

Your First Program

Let's start with the classic first program. In Python, printing text to the screen is just one line:

Python 
print("Hello, World!")
Output
Hello, World!

That's it. One line. No semicolons, no curly braces, no boilerplate. That's the Python difference.

Installing Python

To get started, you need Python installed on your computer:

  1. Visit python.org and download the latest version (Python 3.x)
  2. Run the installer — check "Add Python to PATH" during setup
  3. Open a terminal and type python --version to confirm
💡
Tip

If you want to start immediately without installing anything, try python.org/shell or Google Colab for a browser-based Python environment.

Running Python Code

There are two main ways to run Python:

Interactive Mode

Open a terminal and type python to enter the interactive shell (REPL). Type code and see results instantly — great for experiments.

Shell 
$ python
>>> 2 + 2
4
>>> print("It works!")
It works!

Script Mode

Save your code in a .py file and run it from the terminal:

Shell 
$ python my_script.py

Comments in Python

Comments are lines the Python interpreter ignores — they're notes for humans reading the code. Use # to write a comment:

Python 
# This is a comment — Python ignores it
print("This line runs")  # Inline comment

# Multi-line comments use multiple # symbols
# or triple-quoted strings (used as docstrings)
"""
This is a multi-line string
often used for documentation
"""
⚠️
Common Mistake

Don't confuse comments with code! Anything after # on a line is completely ignored by Python.

Variables

Beginner ⏱ ~7 min read

A variable is a named container that holds a value. Think of it like a labelled box — you give it a name, put something inside, and refer to it by that name later. Variables are the most fundamental building block of any program.

Creating Variables

In Python, you don't need to declare a variable's type beforehand. Just write the name, an equals sign, and the value:

Python 
name = "Alice"
age = 30
height = 5.7
is_student = True

print(name)      # Alice
print(age)       # 30
print(height)    # 5.7
print(is_student) # True
Output
Alice 30 5.7 True

Naming Rules

Variable names in Python must follow these rules:

RuleExample ✅Example ❌
Start with a letter or underscore_name, age1name
Letters, numbers, and underscores onlyuser_ageuser-age
Case-sensitivescore ≠ Score—
Not a reserved keywordvaluefor, if
💡
Convention: snake_case

Python developers use snake_case for variable names — all lowercase with underscores between words. Example: user_name, total_score, is_active.

Multiple Assignment

Python lets you assign multiple variables at once — a handy shortcut:

Python 
# Assign the same value to multiple variables
x = y = z = 0
print(x, y, z)   # 0 0 0

# Assign different values in one line
first, last, age = "Ada", "Lovelace", 36
print(first, last, age)  # Ada Lovelace 36

# Swap two variables (Python's elegant trick)
a, b = 10, 20
a, b = b, a
print(a, b)  # 20 10

Dynamic Typing

Python is dynamically typed — a variable can hold any type of value, and you can reassign it to a different type at any time:

Python 
x = 42           # x is an integer
print(type(x))   # <class 'int'>

x = "hello"      # now x is a string
print(type(x))   # <class 'str'>

x = 3.14         # now x is a float
print(type(x))   # <class 'float'>
⚠️
Be Careful

Dynamic typing is flexible, but reassigning a variable to a completely different type mid-program can make code confusing. Use clear names and be consistent.

Constants

Python doesn't have a built-in constant keyword, but by convention, variables written in ALL_CAPS are treated as constants — values that shouldn't change:

Python 
PI = 3.14159
MAX_USERS = 1000
APP_NAME = "PyLearn"

# Python won't stop you from changing these, but
# the convention signals "don't change this value".

Best Practices

  • Use descriptive names — user_age beats ua every time
  • Keep names lowercase with underscores (snake_case)
  • Constants in ALL_CAPS
  • Avoid single-letter names except for loop counters (i, j)
  • Don't shadow Python built-ins: avoid names like list, str, type

Data Types

Beginner ⏱ ~10 min read

Every value in Python has a type — a classification that tells Python what kind of data it is and what operations are allowed on it. Understanding types is foundational to writing correct, efficient Python code.

Built-in Types Overview

TypeKeywordExampleDescription
Integerint42Whole numbers
Floatfloat3.14Decimal numbers
Stringstr"hello"Text
BooleanboolTrueTrue or False
Listlist[1,2,3]Ordered, mutable
Tupletuple(1,2,3)Ordered, immutable
Dictionarydict{"a":1}Key-value pairs
Setset{1,2,3}Unique, unordered
NoneNoneTypeNoneAbsence of value

Numbers

Python 
# Integers — no size limit in Python!
x = 42
big = 1_000_000   # underscores for readability

# Floats
pi = 3.14159
scientific = 1.5e10  # 15,000,000,000

# Complex numbers
z = 3 + 4j

print(type(x))   # <class 'int'>
print(type(pi))  # <class 'float'>

Strings

Strings are sequences of characters enclosed in single ' or double " quotes. Python strings are immutable — you can't change a character in place.

Python 
greeting = "Hello, World!"
name = 'Python'

# f-strings — the modern way to embed values
age = 30
message = f"I am {age} years old"
print(message)  # I am 30 years old

# Common string methods
print(greeting.upper())   # HELLO, WORLD!
print(greeting.lower())   # hello, world!
print(greeting.split(",")) # ['Hello', ' World!']
print(len(greeting))       # 13

Lists

Lists are ordered, changeable collections that can hold items of any type — including other lists.

Python 
fruits = ["apple", "banana", "cherry"]

# Indexing (0-based)
print(fruits[0])   # apple
print(fruits[-1])  # cherry (last item)

# Modifying
fruits.append("date")    # add to end
fruits.insert(1, "avocado") # insert at index
fruits.remove("banana")  # remove by value

print(fruits)
# ['apple', 'avocado', 'cherry', 'date']

Dictionaries

Dictionaries store data as key-value pairs — like a real dictionary where words (keys) map to definitions (values).

Python 
person = {
    "name": "Alice",
    "age": 30,
    "city": "London"
}

print(person["name"])       # Alice
print(person.get("age"))    # 30

# Add or update
person["email"] = "alice@example.com"

# Loop through
for key, value in person.items():
    print(f"{key}: {value}")
🚫
Common Mistake

Accessing a dict key that doesn't exist raises a KeyError. Use .get(key) to safely get a value, returning None instead of crashing.

Functions

Beginner ⏱ ~12 min read

Functions are reusable blocks of code that perform a specific task. Instead of writing the same logic multiple times, you define it once and call it whenever needed. Functions are the building blocks of clean, maintainable programs.

Defining a Function

Use the def keyword, followed by the function name, parentheses, and a colon. The function body is indented:

Python 
def say_hello():
    print("Hello there!")

# Call the function
say_hello()  # Hello there!

Parameters & Arguments

Functions can accept input values called parameters:

Python 
def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

greet("Alice")              # Hello, Alice!
greet("Bob", "Hey")         # Hey, Bob!
greet(greeting="Hi", name="Charlie")  # Hi, Charlie!

Return Values

Functions return results using the return keyword. A function without a return implicitly returns None.

Python 
def add(a, b):
    return a + b

def min_max(numbers):
    return min(numbers), max(numbers)  # return multiple values!

result = add(3, 7)
print(result)  # 10

low, high = min_max([5, 1, 9, 3])
print(low, high)  # 1 9

Lambda Functions

Lambda functions are compact, anonymous functions defined in a single expression — perfect for short-lived operations:

Python 
# Regular function
def square(x):
    return x ** 2

# Equivalent lambda
square = lambda x: x ** 2

print(square(5))  # 25

# Lambdas shine in sorting and filtering
names = ["Charlie", "Alice", "Bob"]
names.sort(key=lambda n: len(n))
print(names)  # ['Bob', 'Alice', 'Charlie']
💡
When to use lambda

Use lambdas for simple, throwaway functions. For anything more than one expression, define a proper def function — it's clearer and easier to test.

Loops

Beginner⏱ ~10 min read

Loops let you repeat a block of code multiple times. Python has two main loop types: for loops for iterating over sequences, and while loops for repeating while a condition is true.

for Loop

Python
# Loop over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# Loop with range
for i in range(5):
    print(i)  # 0, 1, 2, 3, 4

# enumerate — get index and value
for idx, fruit in enumerate(fruits):
    print(f"{idx}: {fruit}")

while Loop

Python
count = 0
while count < 5:
    print(count)
    count += 1

Object-Oriented Programming

Intermediate⏱ ~15 min read

Object-Oriented Programming (OOP) organizes code around objects — bundles of data (attributes) and behavior (methods). Python is a fully object-oriented language, and understanding OOP unlocks powerful design patterns.

Classes and Objects

Python
class Dog:
    # Class attribute (shared by all instances)
    species = "Canis familiaris"

    def __init__(self, name, age):
        # Instance attributes
        self.name = name
        self.age = age

    def bark(self):
        return f"{self.name} says: Woof!"

    def __repr__(self):
        return f"Dog(name={self.name!r}, age={self.age})"

# Create objects (instances)
buddy = Dog("Buddy", 3)
max_  = Dog("Max", 5)

print(buddy.bark())   # Buddy says: Woof!
print(max_.name)      # Max

Inheritance

Python
class GuideDog(Dog):  # inherits from Dog
    def __init__(self, name, age, owner):
        super().__init__(name, age)  # call parent __init__
        self.owner = owner

    def guide(self):
        return f"{self.name} is guiding {self.owner}"

rex = GuideDog("Rex", 4, "Alice")
print(rex.bark())   # inherited method
print(rex.guide())  # Rex is guiding Alice

File Handling

Intermediate⏱ ~10 min read

Python makes it easy to read and write files. Working with files lets your programs save data, process documents, and interact with the filesystem.

Reading Files

Python
# The 'with' statement handles closing automatically
with open("data.txt", "r") as f:
    content = f.read()        # read entire file
    print(content)

with open("data.txt", "r") as f:
    for line in f:             # read line by line
        print(line.strip())

Writing Files

Python
with open("output.txt", "w") as f:
    f.write("Hello, file!\n")

# Append to existing file
with open("output.txt", "a") as f:
    f.write("Another line\n")
💡
Always use with

The with open(...) pattern ensures the file is properly closed even if an error occurs. Never leave files open manually.

Exception Handling

Intermediate⏱ ~10 min read

Errors are inevitable in programming. Python's exception handling lets your program respond to errors gracefully instead of crashing.

try / except

Python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Can't divide by zero!")

try:
    number = int("not a number")
except ValueError as e:
    print(f"Error: {e}")
finally:
    print("This always runs")

Advanced Python

Intermediate⏱ ~20 min read

Advanced Python features let you write more expressive, efficient, and Pythonic code. This section covers decorators, generators, comprehensions, and context managers.

List Comprehensions

A compact way to create lists from iterables — often replacing multi-line for loops in a single line:

Python
# Traditional way
squares = []
for x in range(10):
    squares.append(x ** 2)

# List comprehension — same result
squares = [x ** 2 for x in range(10)]

# With filtering
evens = [x for x in range(20) if x % 2 == 0]

# Dict comprehension
word_lengths = {word: len(word) for word in ["apple", "fig", "banana"]}

Decorators

Python
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"Took {time.time() - start:.4f}s")
        return result
    return wrapper

@timer
def slow_task():
    import time; time.sleep(0.1)

slow_task()  # Took 0.1001s
